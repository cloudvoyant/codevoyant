# pr — Pull/Merge Request Workflows

| Command | Description |
|---|---|
| `/pr open [--base <b>] [--bug\|--feature] [--title "..."] [--local]` | Create a draft PR/MR from the current branch |
| `/pr review [pr-id] [--name <slug>] [--local]` | Inline review for a PR/MR + a dedicated slop/scope pass |
| `/pr address [pr-id] [--name <slug>] [--local] [--no-resolve]` | Pull comments, apply fixes, respond + resolve threads |
| `/pr update ["change request"] [--type body\|review\|address]` | Apply `<!-- > … -->` annotations or a chat edit to the last artifact |
| `/pr squash [pr-id] [--base <b>] [--onto <n>\|--single] [--no-push]` | Squash branch commits into coherent, changelog-ready commit(s) |
| `/pr publish [pr-id] [--push] [--yes] [--ready-only\|--review-only]` | Publish a draft PR/MR (mark ready) and/or its pending review (alias `/pr ready`) |
| `/pr complete [pr-id] [--event APPROVE\|REQUEST_CHANGES\|COMMENT]` | Publish a pending draft **review** only |
| `/pr help` | Print this reference |

`publish` is the umbrella — it marks the PR/MR ready **and** submits any pending review; `complete` only publishes the review.

## --local — review locally or draft on the platform

`open`, `review`, and `address` **draft directly on the PR/MR by default** — review the agent's work in the GitHub/GitLab UI. Pass `--local` to instead write the work to `.codevoyant/review/{slug}/*.md` and stop, so you can read/edit it locally first (`pr-body.md`, `new-review.md`, or `address.md`). Re-run without `--local` to push it.

## update — edit the last artifact

Adjust whatever `open` / `review` / `address` produced last, from `<!-- > … -->` (minor) / `<!-- >> … -->` (major) annotations you add inline to its local file, or a plain-language request (`/pr update "tighten the summary"`). Re-syncs to the platform if the artifact was already pushed.

## review — slop / scope pass

Alongside the correctness review, a dedicated **slop-detector** agent audits the diff for AI slop and unwanted
agent-introduced change: unnecessary/out-of-scope edits, stochastic churn (random renames, reordering, reformatting),
verbose boilerplate, dead/debug leftovers, and dependency creep. Its findings are prefixed `Slop:` in the review.

## squash

Collapses noisy branch history (WIP / fixup / "fix CI" commits) into one or more
coherent commits before merge. Each result gets a conventional-commit subject and a
changelog-ready body whose lines are hard-wrapped at 72 columns so they never render
truncated. Defaults to one commit; splits into a few only for clearly separate concerns.
Rewrites via soft reset (no interactive rebase) and force-pushes with `--force-with-lease`;
a `pre-squash/<branch>` backup tag is set first. Never adds agent self-attribution.

## Review docs
Stored under `.codevoyant/review/{slug}/`:
- `new-review.md` — generated by `pr review`
- `comments.md` — fetched by `pr address`
- `address.md` — fix proposals from `pr address`

## Requirements
- `gh` CLI for GitHub projects (`gh auth login`)
- `glab` CLI for GitLab projects (`glab auth login`)
