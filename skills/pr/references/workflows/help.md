# rev — Code Review Workflows

| Command | Description |
|---|---|
| `/rev new [pr-id] [--name <slug>]` | Generate AI-powered inline review for a PR/MR |
| `/rev address [pr-id] [--name <slug>]` | Pull comments, propose fixes, apply, respond via draft |
| `/rev complete [pr-id] [--event APPROVE\|REQUEST_CHANGES\|COMMENT]` | Publish pending draft review |
| `/rev help` | Print this reference |

## Review docs
Stored under `.codevoyant/review/{slug}/`:
- `new-review.md` — generated by `rev new`
- `comments.md` — fetched by `rev address`
- `address.md` — fix proposals from `rev address`

## Requirements
- `gh` CLI for GitHub projects (`gh auth login`)
- `glab` CLI for GitLab projects (`glab auth login`)
