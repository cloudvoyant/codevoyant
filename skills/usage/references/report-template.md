## Methodology Summary

Attribution in this report follows the codevoyant usage methodology:

1. **Decision Log entries** (primary): `[user]` and `[agent]` entries recorded during planning and execution
2. **Review rounds**: explicit user instructions to review, annotate, or correct agent output — each counts as a user control assertion
3. **Commit type defaults**: `chore`, `refactor`, `docs`, `test`, `style`, `revert` → user; `feat`, `fix`, `perf` → weighted by Decision Log ratio
4. **Vibecoded classification**: no user decisions + no review rounds + all output accepted without correction

Full methodology: `skills/usage/references/methodology.md`
