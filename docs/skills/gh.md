# gh

GitHub-specific skill — CI monitoring, PR review management, and draft review publishing.

The `gh` skill owns all GitHub platform interactions: watching Actions pipelines, fetching and posting inline PR review comments, and managing draft reviews through their full lifecycle. It provides the primitives that [`/pr`](/skills/pr) delegates to for GitHub projects.

## Installation

**Claude Code:**
```bash
npx skills add cloudvoyant/codevoyant
```

**OpenCode / VS Code Copilot:** See the [installation guide](/user-guide#installation).

## Requirements

- [`gh` CLI](https://cli.github.com/) installed and authenticated (`gh auth login`)
- Active Git repository with a GitHub remote

## Typical Workflows

### Watch CI after a push

```bash
/gh ci                          # watch Actions for current branch
/gh ci --branch feature/foo    # watch a specific branch
/gh ci --autofix               # fix failures and re-push automatically
```

### Run a code review

```bash
/rev new          # generate AI-powered review — calls /gh push-comments internally
/rev address      # fix review comments — calls /gh pull-comments internally
/rev complete     # publish the draft — calls /gh complete internally
```

### Manually manage a PR review

```bash
/gh pull-comments               # fetch unresolved threads to .codevoyant/review/{branch}/comments.md
/gh push-comments               # post inline comments from a review doc to the PR
/gh draft                       # create or update a pending (unpublished) review
/gh resolve-comments            # mark threads resolved
/gh complete                    # publish the pending review
```

## Skills

### CI Monitor

Watch GitHub Actions pipelines for the current branch. Runs in the **background** and sends a desktop notification when checks pass or fail.

```bash
/gh ci
/gh ci --branch <name>    # watch a specific branch (default: current)
/gh ci --autofix          # fix failures and re-push (max 2 attempts)
/gh ci --silent           # suppress desktop notification
```

What happens:
1. Lists recent Actions runs for the branch (last 10 minutes)
2. Spawns a background agent that calls `gh run watch` for each run
3. On failure: fetches logs via `gh run view --log-failed`
4. If `--autofix`: applies fixes, commits, pushes with `--force-with-lease`, re-monitors

> **Migrated from `/git ci`** — If you used `/git ci` before, use `/gh ci` for GitHub projects and [`/glab ci`](/skills/glab) for GitLab.

### Pull Comments

Fetch unresolved PR review threads and write them to a local document:

```bash
/gh pull-comments [pr-number]
/gh pull-comments --output .codevoyant/review/my-feature/comments.md
```

Writes to `.codevoyant/review/{branch}/comments.md` by default. The document lists each thread with: file path, line number, reviewer, comment body, and diff hunk.

### Push Comments

Post inline review comments from a local review document to a PR:

```bash
/gh push-comments [pr-number]
/gh push-comments --doc .codevoyant/review/my-feature/new-review.md
```

Reads the review document (generated by `/rev new` or written manually) and posts each comment as a pending review via the GitHub API. Created review stays in `PENDING` state until you run `/gh complete`.

### Draft

Create or update a pending (unpublished) PR review:

```bash
/gh draft [pr-number]
/gh draft --body "Overall: looks good, one blocking issue"
/gh draft --review-id <id>    # update an existing pending review
```

### Resolve Comments

Mark review threads as resolved via GraphQL:

```bash
/gh resolve-comments [pr-number]
/gh resolve-comments --thread-ids <id1,id2>    # resolve specific threads only
```

### Complete

Publish a pending draft review:

```bash
/gh complete [pr-number]
/gh complete --event APPROVE             # approve the PR
/gh complete --event REQUEST_CHANGES     # request changes
/gh complete --event COMMENT             # comment only (default)
/gh complete --body "LGTM after fixes"
```

### List All Commands

```bash
/gh help
```
